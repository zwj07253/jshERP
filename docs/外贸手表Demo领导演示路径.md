# 外贸手表 Demo 领导演示路径

## 演示主线

这套 Demo 讲的是一条完整外贸链路：

```text
中国采购手表 → 发运到墨西哥 → 清关单证跟踪 → 墨西哥库存 → 分销销售 → 到岸成本与毛利
```

建议演示时间控制在 8-12 分钟，重点不要讲系统功能清单，而是讲“业务闭环”和“管理能看到什么”。

## 1. 外贸驾驶舱

入口：`外贸管理 → 外贸驾驶舱`

讲法：

“这里是外贸业务总览，不只是传统进销存，而是把采购、跨境运输、墨西哥库存、客户订单和毛利放到同一个看板里。”

重点看：

- 外贸采购总额：说明当前 Demo 的采购货值基础。
- 墨西哥锁定销售额：说明已有海外客户订单锁定。
- 墨西哥可销售货值：说明已经到海外仓、还能继续分销的库存。
- 在途风险单证数：说明系统能提前暴露清关/单证风险。
- 墨西哥销售毛利看板：说明销售收入、售出成本、毛利已经能按 SKU 追踪。

## 2. 发运批次

入口：`外贸管理 → 发运批次`

推荐打开：`SH-MX-2026-001`

讲法：

“这个页面从批次角度看履约。一个批次可以关联多个采购 SKU，也可以继续关联海外销售单。这样就能回答：这批货从哪里来、发到哪里、卖给谁、还剩多少。”

重点看：

- 批次状态：已入库、在途、清关中等。
- SKU 明细：采购单、SKU、发运数量、已入库数量。
- 销售单：`MX-SO-20260712-001`
- 客户：`México Tiempo Distribución`
- 销售金额：说明该批货已经和墨西哥分销客户打通。

可以演示的操作：

- 点击“履约详情”。
- 看 SKU 明细里的采购单和销售单。
- 对未关联销售的 SKU 点击“关联销售”，展示可选择销售明细。

## 3. 清关单证

入口：`外贸管理 → 清关单证`

讲法：

“外贸场景里，货不只是库存问题，清关单证会直接影响能不能到仓、能不能卖。这里把 Commercial Invoice、Packing List、Bill of Lading、Pedimento、NOM 文件等放进同一个列表，异常能被提前看到。”

重点看：

- 缺失/审核中状态。
- 责任人。
- 截止日期。
- 异常说明。

建议强调：

“这不是要替代海关系统，而是让 ERP 里能跟踪外贸履约风险。”

## 4. 外贸库存

入口：`外贸管理 → 外贸库存`

讲法：

“传统库存只看仓库数量，但外贸库存要区分海运在途、清关中、海外可销售、客户锁定。否则销售和采购都不知道货到底能不能承诺给客户。”

重点看：

- 状态汇总卡：海运在途、清关中、墨西哥可销售库存、客户锁定库存。
- 明细表：
  - 发运批次
  - SKU
  - 已入库
  - 可销售
  - 客户锁定
  - 销售单
  - 锁定客户

推荐筛选：

- 筛选“墨西哥可销售库存”，说明还有多少能继续卖。
- 筛选“客户锁定库存”，说明哪些已经被客户订单占用。

## 5. 进口成本与销售毛利

入口：`外贸管理 → 进口成本`

讲法：

“这里是领导最关心的结果：这批货卖出去之后到底赚不赚钱。系统把采购成本、物流、关税、本地费用分摊到 SKU，再按已销售/锁定数量折算售出到岸成本，最后算实际毛利和毛利率。”

重点看：

- 发运批次：例如 `SH-MX-2026-001`
- SKU：手表型号
- 客户：`México Tiempo Distribución`
- 销售金额
- 售出到岸成本
- 实际毛利
- 实际毛利率

可以演示的操作：

- 登记一笔费用。
- 选择发运批次后点击“重新分摊”。
- 回到表格查看 SKU 到岸成本和毛利变化。

## 6. 推荐收尾话术

可以这样收尾：

“这个 Demo 第一阶段不是做真实海关接口，而是先把外贸业务结构跑通。它解决的是管理视角：从中国采购开始，到墨西哥清关、海外库存、客户销售和利润核算，业务链路能在 ERP 里闭环呈现。后续如果继续做，可以再接真实汇率、物流轨迹、清关接口和更完整的财务核算。”
