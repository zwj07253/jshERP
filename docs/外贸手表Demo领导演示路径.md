# 外贸手表 Demo 领导演示路径

## 采购—仓库—外贸—销售—零售演示文案

### 开场：先讲清楚这套系统解决什么问题

“今天演示的是一套以手表业务为例的进销存与外贸协同 Demo。整套系统不是把采购、库存、销售做成彼此独立的页面，而是把一只手表从采购进来、进入仓库、发往海外、完成销售，再回到零售端经营的全过程串起来。

我们按照实际业务顺序来看：先采购，采购到货后进入仓库；有海外订单后进入外贸发运和清关；货物到位后完成销售；最后在零售端查看终端销售和经营结果。这样管理层能看到的不只是单据，而是货物、资金和利润如何流转。”

### 第一站：采购——把需求变成可追踪的采购订单

入口：`采购管理 → 请购单 / 采购订单`

“首先从采购开始。业务部门根据销售预测、客户订单和库存情况提出采购需求，采购人员再把需求转成采购订单。以本次手表 Demo 为例，我们采购的是不同型号、不同规格的手表，每个型号都可以继续追踪数量、单价、供应商和采购批次。

采购这一环节最重要的不是录入一张单，而是明确采购依据：为什么采购、采购多少、从谁那里采购，以及这批货后面准备服务哪个市场。后面仓库入库、外贸发运和成本核算，都会沿用这里的采购信息。”

重点讲：

- 采购商品的型号、规格、数量和采购金额。
- 采购单与后续入库单的关联。
- 采购数量是否能覆盖当前订单和安全库存。

过渡话术：

“采购完成以后，下一步不是简单地认为货已经结束，而是要确认货物是否真正到仓、数量是否准确、库存是否可用。”

### 第二站：仓库——确认货物到仓，并形成可用库存

入口：`仓库管理 → 采购入库 / 库存查询`

“采购到货后，仓库根据采购订单办理入库。系统会把采购数量转化为实际库存，并保留商品、批次和数量信息。对于手表这类有多个型号的商品，仓库不仅要看总数量，还要能精确到具体 SKU，避免账面上有库存，但实际缺少客户需要的型号。

到这里，我们可以回答两个问题：第一，采购的货有没有到；第二，现在仓库里到底有多少货可以继续安排。若是普通内销订单，这些库存可以直接进入销售；若是外贸订单，还需要继续进入发运和清关流程。”

重点讲：

- 采购入库数量与采购订单数量是否一致。
- 各手表 SKU 的现有库存、库存金额和批次。
- 哪些库存可以销售，哪些库存已经被订单锁定。

过渡话术：

“如果这批货是供应国内销售，接下来可以直接进入销售；如果是发往海外，就要增加运输、清关、海外仓和进口成本这些外贸环节。”

### 第三站：外贸——把国内库存变成可管理的海外履约链路

入口：`外贸管理 → 外贸驾驶舱`

“这里是整套 Demo 的核心。外贸业务和普通销售最大的区别，是货物从国内仓库到海外客户之间还有一段在途、清关和成本确认过程。系统用发运批次把采购、物流、清关、海外库存和海外销售串联起来。

以墨西哥市场为例，这批手表从中国采购后，按照发运批次发往墨西哥。管理层可以在驾驶舱看到采购金额、已锁定销售金额、墨西哥可销售库存、在途风险单证和预计毛利，从而知道货走到哪里、哪些货已经被客户订走、哪些货还可以继续销售，以及这笔业务最终能赚多少钱。”

接着进入：`外贸管理 → 发运批次`

“在发运批次里，可以继续查看起运港、目的港、柜号、批次状态、发运 SKU 和关联销售单。这样一批货从采购开始，到发往墨西哥，再到分配给海外客户，都有明确的业务依据。”

再进入：`外贸管理 → 清关单证`

“外贸不是货发出去就结束了。Commercial Invoice、Packing List、提单、报关和目的国要求的合规文件，任何一项缺失都可能影响清关和到仓。清关单证页面把这些事项集中管理，领导可以直接看到缺什么、谁负责、什么时候到期，以及当前风险在哪里。”

重点讲：

- 发运批次状态：在途、清关中、已入库。
- 墨西哥销售订单与发运批次的关联。
- 外贸库存中的可销售库存和客户锁定库存。
- 进口成本中的采购、物流、关税和本地费用。

过渡话术：

“外贸环节解决的是货物如何安全、合规地到达目标市场。货物到位并完成成本归集后，下一步就要看销售结果。”

### 第四站：销售——从海外订单看到收入与利润

入口：`销售管理 → 销售订单 / 销售出库`

“进入销售环节后，系统不只是记录卖了多少钱，还要把销售和前面的库存、批次、客户以及成本对应起来。以墨西哥客户订单为例，我们可以看到销售了哪些手表 SKU、销售数量是多少、对应哪个发运批次，以及这批货的销售收入和售出成本。

这样领导看到的是一条完整的结果链：采购形成成本，物流和关税形成到岸成本，销售形成收入，收入减去到岸成本后得到毛利。对于不同 SKU，还可以进一步比较哪些型号更赚钱，哪些型号库存周转更快。”

接着进入：`外贸管理 → 进口成本`

“这里把某个批次或 SKU 的销售收入、采购成本、物流分摊、关税分摊、到岸成本、毛利和毛利率放在一起。它能帮助管理层从‘卖了多少’进一步看到‘实际赚了多少’，也为后续报价和采购决策提供依据。”

重点讲：

- 销售收入与销售数量。
- 售出到岸成本和实际毛利。
- SKU 之间的毛利率差异。
- 客户订单、海外库存和销售出库之间的关联。

过渡话术：

“外贸销售体现的是海外市场的订单和利润，最后再回到日常经营端，看这些手表在零售场景下的销售表现和库存消化情况。”

### 第五站：零售——回到终端经营，验证库存和销售成果

入口：`零售管理 → 零售单 / 零售统计`

“零售端是最贴近经营结果的环节。无论是国内现货销售，还是外贸业务沉淀下来的商品，最终都要通过客户成交形成收入。这里可以按商品、时间和销售渠道查看零售金额、销售数量以及库存变化。

对领导来说，零售页面可以用来验证前面几步的成果：采购的货有没有形成库存，库存有没有被销售消化，哪些手表更受欢迎，销售额和毛利是否达到预期。这样系统就从一笔采购，延伸到了库存、外贸、销售和终端经营。”

重点讲：

- 今日、本月和年度零售销售情况。
- 手表 SKU 的销售数量和销售金额。
- 销售结果对库存余额和补货计划的影响。
- 采购、销售和库存数据是否能够相互印证。

### 收尾：总结五个环节形成的管理闭环

“今天的演示按照采购、仓库、外贸、销售、零售五个环节展开。采购解决货从哪里来，仓库解决货有没有到、还有多少，外贸解决货如何发往海外并顺利履约，销售解决收入和利润，零售则回到终端经营结果。

这套 Demo 的价值不只是把几个模块放在一起，而是让同一批手表在不同业务环节中保持连续：采购单能追到入库，入库能追到发运，发运能追到清关和海外库存，海外库存能追到客户销售，销售最终能落到收入、成本和毛利。管理层可以沿着一条业务链路看清楚货、钱和利润的变化。”

### 领导可能追问时的补充回答

- **这是真实海关系统吗？**
  “当前是 Demo，重点展示 ERP 内部的业务协同和风险跟踪，不替代海关或物流平台。后续可以对接真实物流轨迹、汇率和清关接口。”
- **成本是怎么来的？**
  “以采购成本为基础，再把物流、关税和本地费用按批次或 SKU 分摊，形成到岸成本，再与销售收入计算毛利。”
- **能不能知道哪些货还能卖？**
  “可以。系统会区分在途、清关中、已入库、可销售和客户锁定库存，避免把已经被订单占用的货再次承诺给客户。”
- **后续最值得扩展什么？**
  “可以继续完善真实汇率、物流状态、清关接口、批次成本结算、客户信用和销售预测，让 Demo 逐步具备实际业务系统能力。”

## 演示主线

这套 Demo 讲的是一条完整外贸链路：

```text
中国采购手表 → 发运到墨西哥 → 清关单证跟踪 → 墨西哥库存 → 分销销售 → 到岸成本与毛利
```

建议演示时间控制在 8-12 分钟，重点不要讲系统功能清单，而是讲“业务闭环”和“管理能看到什么”。

## 1. 外贸看板

入口：`外贸管理 → 外贸看板`

讲法：

“这里是外贸业务总览，不只是传统进销存，而是把采购、跨境运输、墨西哥库存、客户订单和毛利放到同一个看板里。”

重点看：

- 外贸采购总额：说明当前 Demo 的采购货值基础。
- 墨西哥锁定销售额：说明已有海外客户订单锁定。
- 墨西哥可销售货值：说明已经到海外仓、还能继续分销的库存。
- 在途风险单证数：说明系统能提前暴露清关/单证风险。
- 墨西哥销售毛利看板：说明销售收入、售出成本、毛利已经能按 SKU 追踪。

## 2. 发运批次

入口：`外贸管理 → 发运批次`

推荐打开：`SH-MX-2026-001`

讲法：

“这个页面从批次角度看履约。一个批次可以关联多个采购 SKU，也可以继续关联海外销售单。这样就能回答：这批货从哪里来、发到哪里、卖给谁、还剩多少。”

重点看：

- 批次状态：已入库、在途、清关中等。
- SKU 明细：采购单、SKU、发运数量、已入库数量。
- 销售单：`MX-SO-20260712-001`
- 客户：`México Tiempo Distribución`
- 销售金额：说明该批货已经和墨西哥分销客户打通。

可以演示的操作：

- 点击“履约详情”。
- 看 SKU 明细里的采购单和销售单。
- 对未关联销售的 SKU 点击“关联销售”，展示可选择销售明细。

## 3. 清关单证

入口：`外贸管理 → 清关单证`

讲法：

“外贸场景里，货不只是库存问题，清关单证会直接影响能不能到仓、能不能卖。这里把 Commercial Invoice、Packing List、Bill of Lading、Pedimento、NOM 文件等放进同一个列表，异常能被提前看到。”

重点看：

- 缺失/审核中状态。
- 责任人。
- 截止日期。
- 异常说明。

建议强调：

“这不是要替代海关系统，而是让 ERP 里能跟踪外贸履约风险。”

## 4. 外贸库存

入口：`外贸管理 → 外贸库存`

讲法：

“传统库存只看仓库数量，但外贸库存要区分海运在途、清关中、海外可销售、客户锁定。否则销售和采购都不知道货到底能不能承诺给客户。”

重点看：

- 状态汇总卡：海运在途、清关中、墨西哥可销售库存、客户锁定库存。
- 明细表：
  - 发运批次
  - SKU
  - 已入库
  - 可销售
  - 客户锁定
  - 销售单
  - 锁定客户

推荐筛选：

- 筛选“墨西哥可销售库存”，说明还有多少能继续卖。
- 筛选“客户锁定库存”，说明哪些已经被客户订单占用。

## 5. 进口成本与销售毛利

入口：`外贸管理 → 进口成本`

讲法：

“这里是领导最关心的结果：这批货卖出去之后到底赚不赚钱。系统把采购成本、物流、关税、本地费用分摊到 SKU，再按已销售/锁定数量折算售出到岸成本，最后算实际毛利和毛利率。”

重点看：

- 发运批次：例如 `SH-MX-2026-001`
- SKU：手表型号
- 客户：`México Tiempo Distribución`
- 销售金额
- 售出到岸成本
- 实际毛利
- 实际毛利率

可以演示的操作：

- 登记一笔费用。
- 选择发运批次后点击“重新分摊”。
- 回到表格查看 SKU 到岸成本和毛利变化。

## 6. 推荐收尾话术
